## ❌ المشكلة — مكتب وساطة يعمل على أدوات مبعثرة

مكتب العقارات آلة سريعة لالتقاط الـ leads وإغلاق الصفقات. ومع ذلك يعمل كل مكتب تقريباً على **خليط من الأدوات لم تُبنَ لتعمل معاً** — والفجوات بينها هي بالضبط حيث يتسرّب المال.

| الألم الذي تعيشه يومياً | ماذا يكلّفك فعلاً |
|---|---|
| 📥 الـ leads تصل عبر **واتساب وفيسبوك والبورتالات والزيارات** — ثم تتبعثر بين الهواتف والدفاتر | leads منسيّة بلا متابعة · تضيع لمن يردّ أولاً |
| 🧩 العملاء في تطبيق، العقارات في آخر، **العمولات في Excel**، المستندات في درايف | لا مصدر حقيقة واحد · **إدخال بيانات مكرّر** دائم · أخطاء |
| 🔁 تتنقّل بين **5–7 أدوات** لا تتحدّث مع بعضها | ساعات تضيع في النسخ · سياق ضائع في الفجوات |
| 🌍 معظم "الحلول" **إنجليزية فقط، أو للوكالات فقط، أو بأسعار enterprise** | البروكر في MENA — *خاصة الفردي* — مُهمَل |
| 👁️ لا رؤية حقيقية لأداء **الفرع / الفريق / الإيجنت** | قرارات بالحدس والذاكرة، لا بالبيانات |
| 💸 العمولات والأقساط تُتابَع **يدوياً** | نزاعات، دفعات ضائعة، إيراد متسرّب |

> **الخلاصة:** leads ضائعة، عمولات غير متتبَّعة، إدارة عمياء، وفريق يقضي وقتاً في *إطعام الأدوات* أكثر من *إغلاق الصفقات*.

## 🆚 المقارنة — استبدل الـ stack بالكامل

معظم المكاتب لا تقارن *CRM بآخر* — بل تقارن **"كومة الأدوات التي أستخدمها الآن" مقابل "منصّة واحدة".** إليك ما يجمعه Est8Core:

| المهمة المطلوبة | تُنجَز اليوم عبر… | **Est8Core** |
|---|---|:---:|
| التقاط leads (واتساب · بورتالات · FB) | واتساب + inbox البورتال + دفتر | ✅ **Leads** مُوحَّد |
| قاعدة بيانات العملاء وتاريخهم | Excel / تطبيق جهات اتصال | ✅ **Contacts** + timeline |
| قوائم العقارات | بورتال + جدول | ✅ **Properties** |
| الصفقات والعمولات والأقساط | جداول Excel | ✅ **Deals** (عمولة/أقساط) |
| إدارة الفروع/الفِرق/الأدوار | لا شيء، أو CRM عام | ✅ **Branches & Teams** + نطاقات |
| التواصل مع العملاء | واتساب شخصي | ✅ **واتساب Inbox** *(تطويرات مستقبلية)* |
| التقارير والأهداف | يدوياً، آخر الشهر | ✅ **لوحات تحكم** *(تطويرات مستقبلية)* |

> **من 6+ أدوات مبعثرة إلى منصّة واحدة متّصلة.**

## 🏗️ البنية والأمان

- **Multi-Tenant بعزل schema لكل مستأجر** على PostgreSQL — بيانات كل عميل معزولة فعلياً.
- **أمان أولاً:** JWT · تشفير كلمات المرور (Argon2id) · مصادقة ثنائية (TOTP) · **طبقة تفويض (RBAC) تُفرَض على كل مسار**.
- **بنية معيارية (Modular Monolith)** حديثة وقابلة للتوسّع.
